这是一份 Git/GitHub 常用指令手册及入门进阶教学，涵盖了从基础同步到文件忽略的核心操作。

### 一、 Git 工作流核心概念

在操作前需理解 Git 的三个区域：
1.  **工作区 (Working Directory)**：你在电脑上直接看到的文件夹。
2.  **暂存区 (Staging Area)**：准备提交的临时区域。
3.  **本地仓库 (Local Repository)**：存储在本地的提交记录。
4.  **远程仓库 (Remote Repository)**：GitHub 上的代码库。



---

### 二、 常用核心指令表

#### 1. 初始化与获取
* `git init`：在当前文件夹初始化一个新的本地 Git 仓库。
* `git clone <SSH地址>`：下载远程仓库到本地。

#### 2. 提交更改
* `git status`：查看当前工作区和暂存区的状态（哪些文件改了还没提交）。
* `git add <文件名>`：将指定文件加入暂存区。
* `git add .`：将当前目录下所有改动加入暂存区。
* `git commit -m "提交说明"`：将暂存区内容正式提交到本地仓库。

#### 3. 远程同步
* `git push origin <分支名>`：将本地提交推送到远程仓库。
* `git pull`：拉取远程仓库最新版本并与本地合并。
* `git fetch`：仅拉取远程更新记录，不自动合并（更安全，方便对比）。

#### 4. 分支管理
* `git branch`：列出所有本地分支。
* `git checkout -b <新分支名>`：创建并切换到新分支。
* `git switch <分支名>`：切换到已有分支。
* `git merge <分支名>`：将指定分支的代码合并到当前分支。

#### 5. 撤销与对比
* `git log`：查看提交历史记录。
* `git diff`：查看工作区与暂存区的代码差异。
* `git checkout -- <文件名>`：放弃工作区的修改（还原到上次提交状态）。

### 三、 教学：如何忽略特定文件 (.gitignore)

在实际项目中，有些文件（如日志、临时缓存、本地配置、包含密码的 `.env` 文件）不需要上传到 GitHub。

#### 1. 创建忽略规则
在项目根目录下创建一个名为 `.gitignore` 的文件（注意文件名以点开头，没有后缀）。

#### 2. 编写忽略规则示例
在 `.gitignore` 文件中输入以下内容：
```text
# 忽略所有 .log 文件
*.log

# 忽略 node_modules 文件夹
node_modules/

# 忽略本地配置文件
.env
config.ini

# 忽略特定的文件夹
/temp/

# 但不忽略该文件夹下的某个特定文件
!/temp/keep.txt

# 忽略所有 .exe 格式的可执行文件
*.exe
```

#### 3. 注意事项
* `.gitignore` 只能忽略**尚未被追踪**的文件。如果一个文件已经提交过 GitHub，再把它写进 `.gitignore` 是无效的。
* **解决方法**：如果想忽略已经上传的文件，需要先从暂存区移除（不删除物理文件）：
    `git rm -r --cached <文件名或文件夹名>`，然后再提交并推送。
