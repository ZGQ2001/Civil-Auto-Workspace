这是一份 Git/GitHub 常用指令手册及入门进阶教学，涵盖了从基础同步到文件忽略的核心操作。

### 一、 Git 工作流核心概念

在操作前需理解 Git 的三个区域：
1.  **工作区 (Working Directory)**：你在电脑上直接看到的文件夹。
2.  **暂存区 (Staging Area)**：准备提交的临时区域。
3.  **本地仓库 (Local Repository)**：存储在本地的提交记录。
4.  **远程仓库 (Remote Repository)**：GitHub 上的代码库。



---

### 二、 常用核心指令表

#### 1. 初始化与获取
* `git init`：在当前文件夹初始化一个新的本地 Git 仓库。
* `git clone <SSH地址>`：下载远程仓库到本地。

#### 2. 提交更改
* `git status`：查看当前工作区和暂存区的状态（哪些文件改了还没提交）。
* `git add <文件名>`：将指定文件加入暂存区。
* `git add .`：将当前目录下所有改动加入暂存区。
* `git commit -m "提交说明"`：将暂存区内容正式提交到本地仓库。

#### 3. 远程同步
* `git push origin <分支名>`：将本地提交推送到远程仓库。
* `git pull`：拉取远程仓库最新版本并与本地合并。
* `git fetch`：仅拉取远程更新记录，不自动合并（更安全，方便对比）。

#### 4. 分支管理
* `git branch`：列出所有本地分支。
* `git checkout -b <新分支名>`：创建并切换到新分支。
* `git switch <分支名>`：切换到已有分支。
* `git merge <分支名>`：将指定分支的代码合并到当前分支。

#### 5. 撤销与对比
* `git log`：查看提交历史记录。
* `git diff`：查看工作区与暂存区的代码差异。
* `git checkout -- <文件名>`：放弃工作区的修改（还原到上次提交状态）。

### 三、 教学：如何忽略特定文件 (.gitignore)

在实际项目中，有些文件（如日志、临时缓存、本地配置、包含密码的 `.env` 文件）不需要上传到 GitHub。

#### 1. 创建忽略规则
在项目根目录下创建一个名为 `.gitignore` 的文件（注意文件名以点开头，没有后缀）。

#### 2. 编写忽略规则示例
在 `.gitignore` 文件中输入以下内容：
```text
# 忽略所有 .log 文件
*.log

# 忽略 node_modules 文件夹
node_modules/

# 忽略本地配置文件
.env
config.ini

# 忽略特定的文件夹
/temp/

# 但不忽略该文件夹下的某个特定文件
!/temp/keep.txt

# 忽略所有 .exe 格式的可执行文件
*.exe
```

#### 3. 注意事项
* `.gitignore` 只能忽略**尚未被追踪**的文件。如果一个文件已经提交过 GitHub，再把它写进 `.gitignore` 是无效的。
* **解决方法**：如果想忽略已经上传的文件，需要先从暂存区移除（不删除物理文件）：

    `直接指出原因：`.gitignore` 只能对**尚未被 Git 追踪**的文件生效。一旦文件经历了 `add`、`commit` 和 `push`，Git 的缓存中就已经存在该文件的追踪记录。此时直接在 `.gitignore` 中补充规则是无效的。
---
**解决逻辑是**：强制将目标文件从 Git 的追踪缓存（暂存区）中移除，使其变为“未追踪”状态，随后 `.gitignore` 的规则就会立刻对其生效。

请在 VS Code 终端中按以下步骤操作：

### 一、 清除文件的追踪缓存

该操作**只会将其从 Git 仓库和追踪列表中删除，你本地的物理文件将完好保留**。

* **如果需要移除的是单个文件：**
    ```bash
    git rm --cached <你的文件名>
    # 示例：git rm --cached config.json
    ```

* **如果需要移除的是整个文件夹（需要加 `-r` 递归参数）：**
    ```bash
    git rm -r --cached <你的文件夹名>
    # 示例：git rm -r --cached temp/
    ```

### 二、 检查状态验证

执行以下命令查看当前 Git 状态：
```bash
git status
```
此时你应该能看到两点变化：
1.  终端会显示该文件已被删除（`deleted: <你的文件名>`），这代表它将从远程仓库被移除。
2.  因为你已经提前配置好了 `.gitignore`，所以该文件**不会**出现在“未追踪的文件 (Untracked files)”列表中。

### 三、 提交更新并推送到 GitHub

将“移除文件追踪”作为一个正式的改动提交，并同步给远程仓库。

```bash
# 提交更改
git commit -m "chore: 停止追踪并在仓库中移除已被忽略的文件"

# 推送至 GitHub（假设当前分支为 main）
git push origin main
```

执行完毕后，刷新 GitHub 网页，该文件或文件夹将从远程仓库中消失。而在你的本地电脑上，文件不仅依然存在，且后续对它的任何修改都不会再被 Git 监控。
