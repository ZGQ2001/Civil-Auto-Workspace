# 工程报告自动化排版矩阵 (Civil Auto Workspace) V2.0

## 📝 项目简介
本项目旨在通过 Python 自动化脚本，解决工程检测/鉴定报告在 Word/WPS 及 PDF 处理过程中的高频排版痛点。
通过将“排版规则（JSON）”与“执行逻辑（Python）”彻底解耦，实现了零代码修改即可动态适配不同报告格式的专业级排版能力，大幅降低机械化加班时间。

**作者:** ZGQ
**核心技术栈:** Python 3, tkinter, pywin32, pypdf, PyMuPDF (fitz), Pillow

---

## 📂 目录架构规范
为了保持项目的整洁与高可维护性，新增功能需严格遵守以下目录存放规则：

* **`01_Input/`**：存放需要处理的原始输入文件（如待排版的 Word、待合并的 PDF）。
* **`02_Core/`**：**核心业务逻辑区**。所有 Python 自动化脚本统一平铺放置于此，统一由 `main.py` 调度。
* **`03_Output/`**：存放脚本处理完成后的输出文件（按格式细分如 `01_doc` 等）。
* **`04_Config/`**：**全局规则控制中枢**。存放 `report_style_config.json` 等配置文件，所有字体、字号、间距标准均在此定义。
* **`98_各种使用手册/`**：存放项目相关的业务规范或操作说明文档。
* **`99_old_JSA/`**：存放历史版本的 WPS JSA 宏代码，仅作归档参考。

## 🛠️ 核心模块说明 (`02_Core`)

1.  **统一控制台**
    * `main.py`: 图形化（GUI）主启动面板，负责异步唤醒以下所有子模块，支持统一调度。
2.  **排版双引擎 (由 JSON 驱动)**
    * `body_format.py`: 报告正文排版引擎。智能识别大纲级别，自动处理各级标题、图表名称及常规正文缩进。
    * `table_format.py`: 报告表格排版引擎。全量规整表格宽度、单元格字体对齐，并自带空值校验标红功能。
3.  **专项纠偏工具包**
    * `bracket_format.py`: 全局括号半全角纠偏工具。精准保护纯数字/字母技术参数，强制锁定标准规范代号。
    * `fix_cross_ref.py`: 交叉引用修复器。自动为域代码追加 `\* MERGEFORMAT`，防止目录更新导致格式崩坏。
4.  **PDF 处理套件**
    * `word2pdf.py`: 支持多线程批量 Word 转 PDF 及多 PDF 合并，自带异常阻断与引擎重启机制。
    * `pdf_coordinate_picker.py`: 可视化 PDF 坐标拾取器。用于在 PDF 页面上快速定位并导出坐标配置。
5.  **底层基建**
    * `file_utils_backup.py`: 全局静默克隆备份模块。提供破坏性操作前的绝对安全熔断保护（兼容 Word 与 Excel 对象）。

## 🚀 部署与使用指南

**1. 环境配置**
请确保本地已安装 Python 环境，并在虚拟环境（`.venv`）中安装项目依赖：
```bash
pip install -r requirements.txt
```
*(依赖清单：pywin32, pypdf, PyMuPDF, Pillow)*

**2. 启动方式**
* **推荐方式**：双击根目录下的 `运行主程序.bat`，脚本将自动激活虚拟环境并启动 `main.py` 图形面板。
* **注意事项**：运行 Word 相关排版脚本前，请确保目标文档处于**已保存且打开**状态，程序会在底层直接与应用进程进行 COM 接口通信。

## 💡 后续开发扩展指南
为项目增加新的自动化功能：
1. 在 `02_Core` 目录下新建独立的 Python 脚本（如 `new_tool.py`）。
2. 在该脚本内部实现异常捕捉与 GUI 弹窗反馈。
3. 若涉及源文件修改，必须在操作前调用 `file_utils_backup.py` 进行静默备份。
4. 打开 `02_Core/main.py`，将新工具添加到 `modules` 列表中，即可在主面板自动生成启动按钮。
